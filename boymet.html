HTML + GPT

Как сделать HTML/JS/CSS-страницу с помощью GPT

1. Что такое HTML, CSS и JavaScript

Перед тем как просить модель писать код, важно понимать «кто за что отвечает».

Роли технологий
Basics

HTML — каркас страницы. Заголовки, абзацы, кнопки, формы, блоки.

CSS — внешний вид. Цвета, шрифты, отступы, размеры, тени, закругления.

JavaScript — логика и интерактивность. Алгоритмы, калькуляторы, показ/скрытие элементов, таймеры, формы, LocalStorage и т.п.

В промпте лучше явно просить: «HTML, CSS и JS — в одном файле». Тогда результат можно сразу сохранить как .html и открыть в браузере.

2. Базовый промпт для генерации страницы

Универсальный шаблон, который можно адаптировать под любой лендинг или прототип.

Стартовый запрос для GPT
Prompt

Подставьте своё описание страницы в квадратные скобки:

Создай код для страницы [описание страницы]. HTML, CSS и JS — в одном файле. Добавь адаптивность под мобильные устройства.

Примеры описания:

  • «лендинг онлайн-курса по английскому»;
  • «одностраничный сайт-портфолио UX-дизайнера»;
  • «страница с таймером обратного отсчёта до события».

Чем конкретнее вы опишете структуру и цель страницы, тем меньше доработок понадобится после.

3. Как быстро протестировать полученный код

Никаких IDE не нужно — достаточно блокнота и браузера.

Пошаговый алгоритм
Practice
  1. Откройте блокнот или любой текстовый редактор (Блокнот, VS Code, Sublime Text и т.п.).
  2. Вставьте сгенерированный GPT-код целиком.
  3. Сохраните файл как index.html или landing.html.
  4. Откройте файл двойным кликом — он запустится в браузере.

Если страница не работает:

  1. Нажмите F12 → вкладка Console (Консоль).
  2. Посмотрите сообщения об ошибках (красный текст с номером строки).
  3. Скопируйте текст ошибки и вернитесь к GPT.
Промпт для исправления
В коде ошибка в консоли: [вставить текст ошибки]. Исправь HTML/JS-код так, чтобы страница работала. Верни полный код страницы целиком.

4. Как быстро проапгрейдить промпт

Сначала ТЗ, потом код — так проще контролировать результат.

Шаг 1. Попросить GPT написать ТЗ
Prompt → ТЗ
Напиши задание для разработки HTML/JS/CSS-страницы [описание структуры страницы, поведение интерактивных элементов].
Шаг 2. Код по ТЗ
ТЗ → код
Сделай HTML/JS/CSS-код страницы на основе этого ТЗ. Весь код — в одном файле. Добавь адаптивность под мобильные устройства.
Код по дизайну
Design → код

Если есть макет (Figma, скриншот), можно ссылаться на него или прикладывать картинку.

Сделай HTML/JS/CSS-код страницы на основе этого дизайна. Сохрани структуру, цвета, шрифты и элементы. Сделай код адаптивным под мобильные устройства.

Важно: картинка даёт только статичное состояние. Весь интерактив лучше описывать словами (см. раздел ниже).

5. Дополнения к промпту: структура, шрифты, цвета

Чтобы дизайн получился ближе к ожиданиям, задайте рамки прямо в запросе.

Структура страницы
Layout
Страница состоит из шапки, трёх буллетов, карусели и кнопки.
Типографика и отступы
Typography
Используй шрифт Roboto. Заголовок — Bold 22px. Остальной текст — 17px. Округления у блоков — 24px. Боковые отступы — 16px. Расстояние между блоками — 16px.
Цветовая палитра
Colors
Палитра: #F3F4F5 — для фонов и теней #000000 — для текста и фонов #7F7F83 — для неактивного текста и обводки.

6. Как описывать логику интерактивных элементов

Модели видят картинку как кадр. Поведение элементов нужно описывать текстом.

Формула описания
Logic

Удобный шаблон:

  • Изначально пользователь видит [X].
  • Если пользователь делает [Y], он получает [Z].
Изначально пользователь видит блок с приветствием и кнопкой «Начать». Если пользователь нажимает «Начать», блок с приветствием скрывается, а показывается форма с полями «Имя» и «Email».

Чем подробнее описана логика, тем меньше придётся «догонять» руками в коде.

7. Работа с картинками

От генерации иконок до использования в коде.

Генерация 3D-иконок
Image prompt

Пример промпта для 3D-иконок в стиле Альфы:

A highly polished 3D illustration. The environment is minimalistic, with soft shadows and ambient occlusion. Materials are smooth and shiny, like rubber, metal or plastic. Bright soft light from top-left. Composition centered, with enough white space. No Background.

Для генерации можно использовать любую модель (в том числе Sora, если она доступна).

Как добавить картинку в код
Assets
  1. Загрузите картинку в репозиторий GitHub:
    • Add file → Upload files;
    • выберите файл (например, beznom.png);
    • нажмите Commit changes.
  2. Опишите использование в промпте:
Слева от текста «Подключить безопасный номер» должна быть картинка beznom.png с разрешением 50x50 px.

GPT вернёт разметку вида:

<img src="beznom.png" alt="Безопасный номер" width="50" height="50">

8. Как и зачем сохранять данные пользователя (LocalStorage)

LocalStorage — встроенное хранилище в браузере, доступное из JS.

Когда это полезно
Concept
  • Пользователь прошёл онбординг → при повторном заходе онбординг не показывается.
  • Пользователь ввёл возраст → он автоматически подставляется при следующем визите.
  • Прошло 30 минут с прошлой сессии → меняется состояние таймера или уведомления.
Промпт для GPT
Добавь всплывающее окно-онбординг, которое показывается только при первом заходе. Сохрани результат в localStorage под ключом "onboarding_done".
Демо. Сохранение возраста
LocalStorage demo

Введите возраст и нажмите «Сохранить» — значение сохранится в localStorage. При обновлении страницы оно подставится автоматически.

9. Как опубликовать страницу через GitHub Pages

Простой способ получить публичную ссылку на ваш HTML-файл.

Шаги
Deploy

Шаг 1. Создать репозиторий

  1. Зайдите на GitHub и нажмите New repository.
  2. Введите название репозитория.
  3. Выберите тип Public.
  4. Нажмите Create.

Шаг 2. Загрузить файлы

  1. Внутри репозитория нажмите Add file → Upload files.
  2. Загрузите файл .html (например, index.html).
  3. Нажмите Commit changes.

Шаг 3. Включить Pages

  1. Перейдите в Settings → Pages.
  2. В разделе Source выберите:
    • Deploy from branch;
    • Branch: main;
    • Folder: /root.
  3. Сохраните настройки.

Шаг 4. Получить ссылку

Через некоторое время появится URL вида:

https://[username].github.io/[название-репозитория]/[название-файла].html

Эту ссылку можно отправлять заказчику, класть в портфолио или использовать как демо-среду для дальнейших экспериментов с GPT.
