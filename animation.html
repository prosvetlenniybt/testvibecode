1. Появление (Enter)

Как мягко показать новый блок на экране: модалка, тултип, карточка.

Fade In
Opacity + slight motion
Использовать для появления модалок, подсказок и небольших карточек.
duration: 220–280ms easing: cubic-bezier(.16,1,.3,1)

Блок появляется с лёгкого увеличения прозрачности и минимального смещения. Главное — без резких скачков: сначала прозрачность, потом финальное положение.

Пример
Tooltip · Fade In
Новый бонус доступен

Пользователь видит подсказку после наведения или первого входа в раздел.

Хорошо работает, когда нужно не отвлекать, а мягко подсветить новый элемент.

.fade-in { opacity: 0; transform: translateY(8px); transition: opacity 220ms cubic-bezier(.16,1,.3,1), transform 220ms cubic-bezier(.16,1,.3,1); } .fade-in.visible { opacity: 1; transform: translateY(0); }
Fade + Slide Up
Toasts, bottom-sheets

Контент появляется снизу с небольшим сдвигом вверх и увеличением прозрачности. Подходит для уведомлений, тостов или панелей, которые выезжают от нижней границы экрана.

duration: 260–320ms easing: cubic-bezier(.25, .8, .25, 1)
Пример
Изменения сохранены

Toast хорошо работает, когда нужно коротко подтвердить действие (сохранение, отправка).

.toast { opacity: 0; transform: translateY(16px); transition: opacity 260ms cubic-bezier(.25,.8,.25,1), transform 260ms cubic-bezier(.25,.8,.25,1); } .toast.visible { opacity: 1; transform: translateY(0); }

7. Геймификация: XP и награды

Как визуально показывать прогресс, уровни и награды, чтобы пользователю хотелось «сделать ещё одно действие».

XP Progress Bar
Persistent Progress

Горизонтальный прогресс с подсветкой текущего уровня и расстояния до следующего.

duration: 600ms
easing: cubic-bezier(.19,1,.22,1)

Хорошая идея — обновлять XP после каждого значимого действия пользователя и визуально показывать приближение к следующему уровню.

Пример
Уровень 3 Осталось 40 XP
+10 XP за действие
Всего XP: 60/100

Такой бар можно показывать в профиле, в хедере или в онбординге, привязав XP к ключевым действиям.

/* HTML: контейнер */ <div class="xp-bar-wrapper"> <div class="xp-bar-track"> <div class="xp-bar-fill"></div> <div class="xp-bar-glow"></div> </div> </div> /* JS-логика (упрощённая) */ const fill = document.querySelector(".xp-bar-fill"); let progress = 0.4; function setXP(value) { progress = Math.max(0, Math.min(1, value)); fill.style.transform = `scaleX(${progress})`; }
Coin Reward + Wallet
Micro-reward

Мини-наградой может быть монетка, полёт которой связывает действие и кошелёк.

duration: 700–900ms easing: cubic-bezier(.19,1,.22,1)

Такой паттерн хорошо работает в обучении, задачах, чек-листах и квестах.

Пример
₵
Баланс: 120
Ещё 30 до следующего бонуса
+5

Монетка вылетает из кнопки и «падает» в кошелёк, усиливая связь между действием и наградой.

/* HTML: кнопку и кошелёк можно расположить в одном контейнере */ <div class="coin-wallet">...</div> <div class="coin-fly">+5</div> /* JS-идея: 1. При клике на кнопку — позиционировать .coin-fly возле кнопки. 2. Через CSS-анимацию смещать её по диагонали к кошельку. 3. По окончании анимации обновить баланс. */
